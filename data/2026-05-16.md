### [10] [Failure-Guided Fuzzing for Hybrid Quantum-Classical Programs](https://arxiv.org/abs/2605.14219)
*Lei Zhang*

Main category: cs.SE

TL;DR: 本文提出了一种针对混合量子-经典程序的失败引导式模糊测试方法，通过两阶段策略（先搜索不收敛种子，再局部模糊电路参数）来暴露故障区域。实验表明，失败引导的局部模糊测试是提升效果的关键，而符号执行种子发现的效果因任务而异。


<details>
  <summary>Details</summary>
Motivation: 现有基于采样的模糊测试在有限预算下难以覆盖混合量子-经典程序中经典优化器与量子电路参数联合空间中的故障区域，亟需更高效的测试方法。

Method: 将混合输入建模为经典优化器超参数与量子电路参数的组合，采用两阶段策略：第一阶段搜索不收敛的种子，第二阶段在种子附近局部模糊测试电路参数。对比了五种预算受限策略。

Result: 失败引导的局部模糊测试是主要改进来源；符号执行种子发现在VQE上有额外收益，但在QAOA上不太稳定。

Conclusion: 复用失败信息是混合量子-经典程序测试的有前景方向，但符号执行种子发现的价值依赖于具体工作负载。

Abstract: Hybrid quantum-classical (HQC) algorithms, such as the Variational Quantum Eigensolver (VQE) and the Quantum Approximate Optimization Algorithm (QAOA), are central to near-term quantum computing but remain challenging to test. Sampling-based fuzzing can expose faulty or non-convergent configurations, but under realistic execution budgets, it may miss failure-prone regions in the joint space of classical optimizer settings and quantum circuit parameters.
  This paper studies failure-guided fuzzing for HQC programs. It models a hybrid input as a pair of classical optimizer hyperparameters and quantum circuit parameters, and evaluates a two-phase strategy that first searches for non-convergent seeds and then locally fuzzes circuit parameters around those seeds. To understand where the gains come from, five budgeted strategies are compared: random hybrid testing, classical enumeration without fuzzing, random-seed local fuzzing, enumeration-seed local fuzzing, and concolic-seed local fuzzing. The study is implemented on a VQE instance and a QAOA MaxCut instance in Qiskit. The results show that failure-guided local fuzzing is the main driver of improvement over random testing, while concolic seed discovery provides additional benefits on VQE but is less stable on QAOA. These findings suggest that reusing failure information is a promising direction for HQC testing, but that the value of concolic seed discovery is workload-dependent.

### [11] [Remember Your Trace: Memory-Guided Long-Horizon Agentic Framework for Consistent and Hierarchical Repository-Level Code Documentation](https://arxiv.org/abs/2605.14563)
*Suyoung Bae,Jaehoon Lee,Changkyu Choi,YunSeok Choi,Jee-Hyong Lee*

Main category: cs.SE

TL;DR: MemDocAgent是一个长周期智能体框架，通过依赖感知遍历引导和内存引导的智能体交互，在单一集成上下文中为整个代码仓库生成结构化文档，性能优于现有开源和闭源基线。


<details>
  <summary>Details</summary>
Motivation: 现有仓库级文档生成方法独立处理各组件，导致检索冗余、文档间描述冲突，且输出缺乏层次化结构。

Method: 提出MemDocAgent框架，包含两个组件：(i) 依赖感知遍历引导——预先确定遵循依赖和粒度层次结构的遍历顺序；(ii) 内存引导的智能体交互——智能体通过读、写、验证操作与共享内存RepoMemory交互，累积先前工作痕迹。

Result: 在多标准深入评估中，MemDocAgent在开源和闭源基线方法上均取得了最佳性能，并在实际软件开发工作流中展现了实用性。

Conclusion: MemDocAgent通过集成上下文和记忆机制有效解决了仓库级文档生成的冗余和冲突问题，具有实际应用价值。

Abstract: Automated code documentation is essential for modern software development, providing the contextual grounding that both human developers and coding agents rely on to navigate large codebases. Existing repository-level approaches process components independently, causing redundant retrieval and conflicting descriptions across documents while producing outputs that lack hierarchical structure. Therefore, we propose MemDocAgent, a long-horizon agentic framework that generates documentation within a single, integrated context spanning the entire repository. It combines two components: (i) Dependency-Aware Traversal Guiding that predetermines a traversal order respecting dependency and granularity hierarchies; (ii) Memory-Guided Agentic Interaction, in which the agent interacts with RepoMemory, a shared memory accumulating prior work traces through read, write, and verify operations. Through an in-depth multi-criteria evaluation, MemDocAgent achieves the best performance over both open and closed-source baselines and demonstrates practical applicability in real software development workflows.

### [12] [Veritas: A Semantically Grounded Agentic Framework for Memory Corruption Vulnerability Detection in Binaries](https://arxiv.org/abs/2605.15097)
*Xinran Zheng,Alfredo Pesoli,Marco Valleri,Suman Jana,Lorenzo Cavallaro*

Main category: cs.SE

TL;DR: Veritas是一个语义驱动的二进制内存损坏漏洞检测框架，结合静态切片、双视图LLM检测器和多智能体验证器，在真实二进制漏洞基准上实现90%召回率且几乎无假阳性，并发现了一个新的Apple CVE漏洞。


<details>
  <summary>Details</summary>
Motivation: 二进制内存损坏漏洞检测需要从低层级、信息丢失的表示中恢复对象语义、过程间传播和可行触发条件，现有基于LLM的方法缺乏内存相关语义和运行时可行性证据的可靠支撑。

Method: Veritas由三部分组成：(1) 静态切片器：基于RetDec提升的LLVM IR重建值流关系，生成紧凑的见证流对象；(2) 双视图LLM检测器：结合反编译C和选择性LLVM IR，逐步推理受控流、边界和对象对应关系；(3) 多智能体验证器：通过引导式调试、断点检查和内存校验来确认或拒绝候选漏洞。

Result: 在真实二进制漏洞基准上召回率达90%；623个候选漏洞的穷举验证产生0个假阳性，额外审计发现2个确认的假阳性；在实际应用中发现了此前未知的Apple漏洞并获得CVE编号。

Conclusion: 语义接地（semantic grounding）是构建实用二进制漏洞检测系统的有效操作设计原则，能够在低层级二进制表示上实现高精度、低假阳性的漏洞检测。

Abstract: Detecting memory corruption vulnerabilities in stripped binaries requires recovering object semantics, interprocedural propagation, and feasible triggers from low-level, lossy representations. Recent LLM-based approaches improve code understanding, but reliable detection still requires grounding in memory-relevant semantics and runtime feasibility evidence. We present Veritas, a semantically grounded framework for binary memory corruption vulnerability detection. Veritas combines a static slicer over RetDec-lifted LLVM IR, a dual-view LLM detector that reasons step by step over grounded flows using decompiled C and selective LLVM IR, and a multi-agent validator that checks hypotheses against debugger-visible artifacts and runtime evidence. The slicer reconstructs value-flow relations from LLVM-IR facts, including def-use, calls, returns, globals, and pointer operations, and emits compact witness-backed flow objects. The detector uses these artifacts to reason about control flow, bounds, and object correspondence without rediscovering whole-binary propagation. The validator confirms or rejects candidates through guided debugging, breakpoint inspection, and memory-checking oracles. We implement Veritas as a modular pipeline and evaluate it on a curated benchmark of real-world binary vulnerability cases. Veritas achieves 90\% recall. For false-positive assessment, we exhaustively validate and manually verify 623 detector candidates and audit additional candidates from larger cases. The exhaustive subset produces no false positives, while the additional audit identifies two confirmed false positives. In a real-world application, Veritas discovered a previously unknown Apple vulnerability that was confirmed and assigned a CVE. These results support semantic grounding as an operational design principle for practical binary vulnerability detection.

</details>
